# UDPTasks

## Завдання 1: Дослідження активних UDP з'єднань

Які стани ви бачите для UDP сокетів? Чому там немає "LISTENING" або "ESTABLISHED"?
-

Тому що по протоколу UDP байдуже на те чи прийдуть ВСІ пакети, чи не прийдуть. Йому головне передати інформацію.


Які сервіси у вашій системі використовують UDP? (наприклад, порти 53, 123, 1900)
-

3544, 3702, 5050, 5353, і т.д

Чому браузер майже не використовує UDP для звичайного серфінгу (до появи HTTP/3)?
-

Бо браузері важливо передати ВСЮ інформацію, а UDP може частково не докинути інформацію


## Завдання 2: UDP порти та well-known сервіси

Доповніть таблицю популярними UDP портами:
-

| Порт | Сервіс | Призначення |
|---|---|---|
| `53` | DNS | Перетворення доменних імен у IP-адреси |
| `67/68` | DHCP | Автоматична видача IP-адрес |
| `69` | TFTP | Просте передавання файлів |
| `123` | NTP | Синхронізація часу |
| `137/138` | NetBIOS | Служби Windows-мереж |
| `161/162` | SNMP | Моніторинг мережевого обладнання |
| `443` | QUIC / HTTP/3 | Веб-трафік через UDP |
| `500` | IKE | VPN-з’єднання |
| `514` | Syslog | Передача системних логів |
| `1900` | SSDP | Пошук пристроїв UPnP |
| `3544` | Teredo | IPv6 через IPv4 |
| `3702` | WS-Discovery | Пошук пристроїв у Windows |
| `5353` | mDNS | Локальне DNS без сервера |
| `5355` | LLMNR | Локальне визначення імен |


Чому DNS використовує переважно UDP? В яких випадках він переходить на TCP?
-

Для швидкого збереження даних. Запит-відповідь вміщується в одну дейтаграму.

Як працює DHCP, якщо у клієнта ще немає IP-адреси? (Підказка: Broadcast)
-

DHCP працює через broadcast повідомлення. Навіть якщо клієнт ще не має IP-адреси, він надсилає повідомлення на адресу 255.255.255.255

Що таке QUIC і яке відношення він має до UDP?
-

QUIC - Мережевий протокол, який використовується в HTTP/3. Він працює поверх UDP


Завдання 3: Експерименти з надсиланням UDP пакетів
-


Що станеться, якщо ви надішлете UDP пакет на порт, який ніхто не слухає? Чи отримає відправник якусь помилку?
-

Якщо UDP-пакет буде надісланий на порт, який ніхто не слухає, то пакет просто буде відкинутий операційною системою. UDP не гарантує доставку даних.

Чи зберігається порядок повідомлень, якщо надіслати 10 пакетів поспіль через nc?
-

Порядок не гарантується. По протоколу доставка не контролюється, та не відновлюється втрата.
